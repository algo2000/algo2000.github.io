Todo App 작성

Todo List 화면에 표시하기

먼저 미리 작성된 todo 리스트를
화면에 보여줍니다.

src/Todo/Todo.tsx

src/Todo/index.tsx

설명

Todo에서 사용하는 타입입니다.
TodoType은 index.tsx와 공유하기 위하여 export를 하였습니다.
TodoProps는 Todo함수의 Props를 타입으로 가지고 있습니다.

todo 한개를 Props로 받습니다.
해당 Props의 타입을 앞서 만들어준 TodoProps로 해주었습니다.
TodoProps는 todo의 타입을 정의하고 있습니다.
todo의 타입인 TodoType는 text라는 string 타입을 가지고 있습니다.

useState의 todos는 TodoType배열입니다.
타입 정의에 따르면 text는 string 타입입니다.

todos를 map을 통해 하나씩 돌면서 함수를 수행합니다.
여러번 Todo를 사용하기 때문에 해당 컴포넌트 각각에 key값으르 대입해줘야합니다.
수행하는 함수의 파라미터는 todo, index가 있습니다.
todo에는 todos의 하나하나의 값이 차례대로 들어오게 됩니다.
현재 todos의 배열의 크기가 3이니 총 3번 Todo를 돌게 됩니다.
todo={todo}는 Props로 todo를 전달합니다.

Todo 리스트 입력 추가

현재 코드는 코드에 이미 작성된 Todo 리스트를 보여줍니다.
그래서 Todo를 직접 추가할 수 있는 기능을 추가합니다.

src/Todo/TodoForm.tsx

src/Todo/index.tsx

설명

해당 함수의 Props의 타입을 정의합니다.
addTodo라는 Function을 가지고 있습니다.

input 태그의 text의 상태를 저장합니다.

Submit 시에 작동되는 함수입니다.
e.preventDefault()는 이벤트를 취소할 수 있는 경우,
이벤트의 전파를 막지않고 그 이벤트를 취소합니다.
value의 값이 빈값인 경우 값이 추가되지 않습니다.
addTodo()는 todo를 추가하는 행위를 하는 함수입니다.
value의 값을 추가해줍니다.
마지막으로 value의 상태를 초기화 해줍니다.

Submit이벤트가 발동시 handleSubmit를 실행합니다.
value={value}를 통해 input의 text와 value의 상태가 공유됩니다.
onChange이벤트 즉 값이 바뀔경우 이벤트가 터지면서
value의 상태를 입력값으로 갱신합니다.

Todo 리스트에 값을 추가하기 위한 콜백 함수 입니다.
[…todos, { text }]는 기존의 리스트에 추가합니다.
setTodos를 통해 todos를 갱신합니다.

TodoForm의 Props로 addTodo함수를 보냅니다.

Todo의 완료/삭제 기능

마지막으로 완료와 삭제기능을 추가합니다.

src/Todo/index.tsx

src/Todo/Todo.tsx

설명

완료 처리의 상태를 확인하기 위해 isCompleted를 추가로 선언합니다.
추가하기 위해서 TodoType타입에도 isCompleted의 타입을 추가해줘야합니다.

완료 버튼 클릭시 작동하는 콜백 함수입니다.
현재 Todo리스트의 클릭한 항목의 isCompleted의 값의 반대값을 대입해줍니다.
변경된 리스트를 setTodos로 갱신해줍니다.

삭제 버튼 클릭시 작동하는 콜백 함수입니다.
현재 Todo리스트의 클릭한 항목을 삭제해즙니다.
변경된 리스트를 setTodos로 갱신해줍니다.

완료와 삭제시 사용되는 콜백함수의 파라미터로 사용하기 위한 index를 넘겨줍니다.
completeTodo 함수와 removeTodo 함수를 넘겨줍니다.

수정된 코드에 맞게 타입 또한 수정해줍니다.

style으르 추가하여 완료상태에 따른 스타일을 줘서 구분을 해줍니다.
