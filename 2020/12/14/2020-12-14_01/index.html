github pages를 이용한 블로그 생성과 테마 적용하기

Author: Leo Date: 12월 14, 2020  12:31:12 Category: 개발 일지 github pages

Github Pages 만들기

Github Pages 에서 페이지 생성에 대한 내용을 볼 수 있습니다.

git repository 생성

Github에서 repository를 생성합니다.

repo 생성 시 이름 규칙에 주의 합니다.

나머지는 모두 default로 한 후 repo를 생성합니다.

생성된 git repo를 로컬로 clone

생성된 git repo를 로컬로 가져옵니다. 원하는 파일 위치에 가져오면 됩니다.

Hello World를 page에 띄우기 - index.html 생성

로컬에 생성된 repo 폴더안에 들어가 index.html을 작성합니다.

Hello World를 page에 띄우기 - git commit, push

생성된 index를 커밋하고 Github에 push 합니다.

작성자 본인은 기본 branch가 master 입니다.

성공

해당하는 url에 접속하여 Hello World가 보인다면 성공입니다.

Github Pages 에 Hexo를 이용한 테마 적용

Hexo는 정적 페이지를 쉽게 만들 수 있도록 도와주는 서비스 입니다. 여러 테마를 지원하고 있으며 자바스크립트 기반입니다.

Hexo 사용 준비 및 설치

Hexo를 사용하기 위해서는 node.js와 git이 설치 되어있어야 합니다.

해당 설치는 원하는 폴더 안에 하시면 됩니다.
정상적으로 완료가 되면 blog 폴더 안에 각종 파일이 생성됩니다.

이런 식으로 각종 파일이 생성됩니다.

_config.yml 파일 수정

_config.yml 파일은 Hexo 설정 파일 입니다. 해당 파일을 열어 정보를 수정해야합니다.

해당 설정이 모두 완료 되면 저장하고 나옵니다.
세부적인 설정은 따로 찾아보시길 바랍니다.

hexo-deployer-git 플러그인 설치

git과 연동될 수 있도록 플러그인을 설치해 줍니다.

로컬 서버 구동

git과 연동되지 않은 상태에서 서버를 열어 봅니다.

터미널을 열어둔 상태에서 http://localhost:4000에 접속 하여 제대로 나오는지 확인합니다.

Github에 Hexo 배포하기

_config.yml에 설정이 제대로 되어 있다면 git에 commit과 함께 배포가 가능합니다.

배포가 완료된 후 자신의 사이트에서 배포가 정상적으로 되었는지 확인하면 됩니다. 배포가 완료되기 까지의 텀 이 있기 때문에 접속하였을때 안뜨거나 바뀌지 않은 경우 좀만 기다려 주시면됩니다.

테마 적용

우선 Hexo 테마 사이트에 접속하여 원하는 테마를 선택합니다.
테마를 적용하는 방법은 각각의 테마 README에 작성 되어있습니다.

가져온 테마를 적용하기 위해서 _config.yml의 theme값을 변경해야합니다.

theme: [테마 이름]
# ex
theme: Chic

변경을 완료했다면 저장을 합니다.
테마를 적용하기 위해서 Repo 상태를 Clean 한 후에 재배포 합니다.

# repo 상태 clean
$ hexo clean
# 배포
$ hexo deploy --generate

테마 적용 완료


chic이라는 테마를 적용한 모습입니다.
