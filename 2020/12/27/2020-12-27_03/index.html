Author: Leo Date: 12월 27, 2020  18:35:40 Category: 개발 일지 typescript

비동기 콜백 함수

동기, 비동기 API

단일 스레드와 비동기 API

js는 단일 스레드에서 동작하므로 동기 API는 사용하면 안됩니다.
ts 또한 단일 스레드에서 작동하므로 항상 비동기 방식으로 동작하는 API를 사용해 프로그램의 반응성으르 훼손하지 말아야합니다.

콜백 지옥

이처럼 콜백 구조가 얽힌 상태를 콜백 지옥이라고 합니다.
Promise는 이런 콜백 지옥에 빠진 코드를 좀 더 다루기 쉬운 형태의 코드로 만드려는 목적으로 만들어졌습니다.

Promise

js에서는 Promise라는 클래스를 지원 합니다.

ts에서의 Promise는 제네릭 클래스 형태로 사용합니다.

Promise의 콜백함수는 resolve, reject 두개의 매개 변수를 가집니다.

resolve, reject 함수

코드 실행 결과
resolve 함수를 호출한 값은 then 메소드의 콜백 함수 쪽에 전달됩니다.
reject 함수를 호출한 값은 catch 메소드의 콜백 함수 쪽에 전달 됩니다.
finally 메소드는 항상 마지막에 호출됩니다.

then 체인

Promise 객체에 then 메소드를 여러번 호풀하는 것을 then-체인 이라고 합니다.

Promise.all

Array 클래스는 every라는 이름의 메소드를 제공합니다.
every 메소드는 배열의 모든 아이템이 어떤 조건을 만족하면 true를 반환합니다.

ex

Promise 클래스는 every 메소드처럼 동작하는 all이라는 정적 메소드를 제공합니다.
all 메소드는 reject가 하나라도 날 경우 앞서 resolve가 된 내용도 전부 출력되지 않고 catch 메소드로 넘어 값니다.

ex

Promise.race

Array 클래스는 some이라는 메소드를 제공합니다.
some 메소드는 배열의 내용 중 하나라도 조건을 만족하면 true를 반환합니다.

ex

Promise.race 클래스 메소드는 promise 객체 중 하나라도 resolve되면 모든 객체를 반환합니다.
하지만 만약 reject값이 가장 먼저 발생하면 reject객체를 반환합니다.

await
